# V20 1.0 Release Hardening Spec

Status: planned; not implemented.

## Research And Prior Art

Before DWM can be treated as a real product, it needs compatibility guarantees,
security boundaries, migration policy, and reproducible acceptance tests. This
is the point where the project stops adding major surfaces and hardens them.

## Product Position And Non-Goals

V20 is the 1.0 hardening and release-candidate phase.

Non-goals:

- do not add new runtime features during hardening,
- do not weaken safety gates for demo convenience,
- do not declare 1.0 from a single dogfood run,
- do not require OMX to pass acceptance.

## Workflow Architecture

1.0 acceptance covers:

- Core plan and packet contracts,
- Runner evidence contracts,
- session/worktree runtime,
- fanout/fanin,
- review/repair,
- dashboard approvals,
- adapters,
- install and migration.

## Execution Model

Release candidates run a fixed corpus of tracked fixtures plus at least one
real-world dogfood workflow. Every generated summary must be reproducible from
tracked inputs or explicitly marked as optional live smoke evidence.

## Safety And Verification Gates

1.0 cannot pass if any known path allows stale evidence, untracked approval,
silent worktree mutation, hidden backend state, unbounded retry, or unchecked
external action.

## Evaluation Fixtures

- full clean dogfood workflow,
- interrupted/resumed workflow,
- multi-worker fanout workflow,
- human-gated workflow,
- adapter failure workflow,
- migration from V11 artifacts.

## Release Plan

1. Freeze schema compatibility policy.
2. Add migration and rollback docs.
3. Run full fixture corpus and reviewer gate.
4. Publish 1.0 decision only when every required fixture passes.
